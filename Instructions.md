## Overview and Requirements
For this final assignment, you are required to create a solution, as a team (you may have a team of up to 3 members), to a game or graphical application of your choice! Some game possibilities are listed below:

- Throw a ball at a target or snowman and maybe make paint splash later  

You must apply inheritance and polymorphism in your solution. Also, I encourage you to try to use Exception handling, templates, operator overloading, and things you have learned this semester.

Your goal for the assignment is to build a complete graphical game or application. As a team, you must ultimately decide how you will implement graphics. You have many tools and library options available to implement the graphics portion of the assignment. Please be sure to also add some directions on how to play the game or use your application.

Aside from the requirements listed in the above paragraph, you are free to complete this assignment as you see fit. ***During our last lab, the week before Finals, you will have to present your application to the class.*** 

Have fun with this assignment!

I recommend that you read [How to Plan and Build a Programming Project - A Legitimate Guide for Beginners](https://www.peterlunch.com/blog/how-to-plan-and-build-a-programming-project).

## Project Details
### **50 points** - Five weekly summaries (10 pts. each)
### **100 points** - Design and implementation of main gameplay or application
You must apply inheritance and/or polymorphism! Other topics covered in our course can also count towards this (QuickSort, Linked List, Binary Trees, etc). You can also apply more advanced data structures.
### **75 points** - Use of Graphics
### **25 points** - Project Presentation (May 4, 2023)
### **25 points** - Team Evaluation and Self-Evaluation
Each member must conduct a self-evaluation as well as must submit an evaluation of each team member. You have to score yourself and your team members based on the following criteria:
- How well did the member communicate with the team?
- Member participation in meetings?
- Contribution to the project, like ideas, code implementation? Each member must equally contribute to the project
- Is the member a good team player?
### **25 points** - Coding style/documentation
Appropriate design, coding style, and comments according to class standards. Follow [CS150/CS151 Style Guide for C++](https://github.com/nic-cs150-master/style-guide).  
- Elegant code 
  - **refactoring** code from 10 to 3 lines of code
  - use of **functions** to reuse repetitive tasks
  - use of **classes** to encapsulate objects
  - Use of proper **naming conventions** of variable names, function names, class names, etc.
  - **Comments**: File header comments, function header comments, in-line comments
### **25 points** - Creativity and/or originality 
of the game/application and its implementation
### **25 points** - Write a README file in GitHub. 
Might have to learn [Markdown Syntax](https://guides.github.com/features/mastering-markdown/) for good formatting.  
Read this [website for suggestions](https://guides.github.com/features/mastering-markdown/) on a good README file for a programming project.
